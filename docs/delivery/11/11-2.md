## Description

Deliver end-to-end alert experience for recurring transactions: server-side detection of overdue and amount-change scenarios, REST endpoint to expose alerts, and UI to render them.

## Requirements

- `RecurringManager` provides missing and amount change alert helpers.
- `/recurring/api/alerts` returns structured response with severity and metadata.
- `/recurring/api/stats` includes alert counts.
- `/recurring` page shows alert cards with severity styling.
- Alerts clear once new instance recorded (pending task 11-3).
