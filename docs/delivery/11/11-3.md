## Description

Ensure new transactions that match existing recurring patterns automatically create instances and update `next_expected_date`, keeping alerts accurate and preventing false positives.

## Requirements

- During CSV import and manual entry, call `RecurringDetector.check_new_transaction`.
- If match found, create instance via `RecurringManager.add_instance`.
- Update recurring record’s `last_transaction_date` and `next_expected_date`.
- Respect user overrides (e.g., paused recurring entries).
- Log outcome for debugging.

## Implementation Plan

1. Inject detector/manager into import and manual add flows.
2. After transaction persisted, attempt match and create instance.
3. Handle edge cases (multiple matches, paused recurring patterns).
4. Update documentation for import pipeline.

## Test Plan

- Import CSV containing recurring payment and verify instance added.
- Confirm alert count decreases after instance creation.
- Ensure paused recurring items do not create new instances.
- Add unit/integration tests for pipeline.
