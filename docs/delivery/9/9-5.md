## Description

Wire recurring pattern insights into the categorization pipeline so that newly imported transactions automatically inherit the category associated with their recurring pattern. This avoids repeated manual categorization for subscriptions and other predictable payments.

## Requirements

- Invoke `RecurringDetector.check_new_transaction` during CSV import and manual entry flows.
- When a recurring match is found, assign the recurring pattern's category to the transaction prior to persistence.
- Record instance linkage via `RecurringManager.add_instance` to keep next-expected dates accurate.
- Avoid overriding user-set categories unless user opts in.
- Log integration steps for troubleshooting (e.g., matched recurring ID, applied category).

## Implementation Plan

1. Inject recurring detector/manager into import pipeline (`src/routes/import_routes.py`).
2. After deduplication/validation, call `check_new_transaction` for each new transaction.
3. If matched, set `transaction.category_id` and persist recurring instance link.
4. Update categorization engine to skip duplicate rule creation when recurring match exists.
5. Add configuration flag or environment toggle for hard/soft auto-assignment.

## Test Plan

- Import CSV containing known recurring merchant (e.g., Netflix) and verify category auto-applied.
- Confirm recurring instance created and `next_expected_date` updated.
- Validate manual override preserves user category and disables future auto updates if desired.
- Ensure unmatched transactions follow existing categorization engine workflow.
- Add unit/integration tests walking through import -> recurring match -> category assignment.
