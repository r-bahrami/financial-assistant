## Description

Deliver a dedicated recurring transactions management page that surfaces detected patterns, upcoming payments, and actionable alerts. The UI must expose scanning, pause/resume, and delete affordances while presenting key metrics (counts, totals, alerts) for quick comprehension.

## Requirements

- Render recurring statistics (active count, monthly total, upcoming count, alert count).
- Display alert panel highlighting missing payments and amount changes.
- List recurring transaction cards with frequency badges, confidence score, and quick actions.
- Provide status filter (all, active, paused, cancelled).
- Expose `Scan for New Patterns` CTA that triggers backend scan endpoint.
- Ensure empty state messaging when no recurring transactions exist.

## Implementation Plan

1. Create `src/templates/recurring.html` with stats grid, alerts section, and recurring list.
2. Build client-side JS to fetch stats (`/recurring/api/stats`), alerts (`/recurring/api/alerts`), and recurring list (`/recurring/api/all`).
3. Wire buttons for scan, pause, resume, and delete using existing REST endpoints.
4. Update navigation (`base.html`, `index.html`) to link to `/recurring`.
5. Register blueprint in `src/app.py` (if not already).

## Test Plan

- Load `/recurring` and verify stats, alerts, and list render without console errors.
- Trigger scan and verify success modal plus refreshed stats/list.
- Pause/resume/delete actions update card state and refresh metrics.
- Status filter swaps between active/paused/cancelled entries.
- Empty state renders when no recurring transactions available.
