## Description

Provide a command-line utility that analyzes historical transactions, detects recurring patterns, and persists them to the database with user confirmation. This enables users to bootstrap recurring tracking after importing prior statements.

## Requirements

- Support optional `--account-id` filter and `--min-confidence` threshold.
- Present detected patterns with merchant, frequency, average amount, confidence, and occurrence count.
- Prompt user confirmation before saving patterns.
- Persist recurring records and associated instances on confirmation.
- Exit gracefully on cancellation or errors (non-zero status, messaging).

## Implementation Plan

1. Create `src/scan_recurring.py` importing `RecurringDetector`.
2. Parse CLI arguments via `argparse`.
3. Call `detector.detect_patterns(...)` and display summary table.
4. Persist patterns using `detector.save_recurring_pattern`.
5. Handle keyboard interrupts and exceptions with friendly output.
