## Description

Deliver actionable alerts that notify users when expected recurring payments are missing or when the amount deviates beyond tolerance. Alerts should surface in the UI (`/recurring`) and feed dashboard metrics to drive proactive follow-up.

## Requirements

- Detect missing payments when `next_expected_date` is >3 days overdue (configurable).
- Flag amount changes when actual payment differs from average by >10% (configurable).
- Provide severity (e.g., high when >7 days late or >25% change).
- Surface alert counts in `/recurring/api/stats` and list details in `/recurring/api/alerts`.
- Ensure alerts clear automatically once a new instance is recorded.

## Implementation Plan

1. Extend `RecurringManager` with `get_missing_payments`, `get_amount_changes`, and `get_all_alerts`.
2. Update `/recurring/api/alerts` endpoint to return alert payload.
3. Display alerts in template with severity styling and merchant info.
4. Hook import pipeline to create recurring instances so alerts stay accurate.
5. Add configuration constants for overdue days and variance thresholds.

## Test Plan

- Seed recurring pattern with expected dates and simulate overdue scenario; verify missing alert emitted.

- Ensure alerts drop when new instance recorded.
- Verify `/recurring` stats card increments/decrements as alerts appear/disappear.
- Write integration test covering import -> recurring match -> alert resolution.
