## Description

Build API endpoints and UI components that allow users to create, view, update, and delete savings goals, leveraging the foundation established in tasks 8-1 and 8-2.

## Requirements

- REST endpoints (list, detail, create, update, delete) with validation.
- UI for listing goals, showing progress, and editing goals.
- Progress calculation display (target vs. current amount).
- Error handling and user feedback for invalid inputs.
- Integration with budgets/dashboard when appropriate.

## Implementation Plan

- Introduce a dedicated `goals` blueprint that exposes REST endpoints (list/create/update/delete) with computed progress metrics.
- Enhance the `Goal` model and test fixtures to support single-goal retrieval, defaults, and status handling.
- Build a `goals.html` management page featuring summary cards, responsive goal cards, progress bars, and modal-based CRUD forms.
- Register the blueprint and navigation entry so the feature is available through the primary UI shell.
- Add integration coverage to validate API behaviour and guard against regressions.

## Test Plan

- Call `/goals/api` to confirm list + summary payload accuracy.
- Exercise POST/PUT/DELETE flows to verify status transitions, validation, and progress math.
- Assert validation errors return 400 + descriptive messages.
- Run the full `pytest` suite to ensure no regressions across existing features.
